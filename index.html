Qué necesitas

Para asegurarnos de que aproveches al máximo el curso, debes tener conocimientos en HTML, CSS y JavaScript. No es necesario que seas un gurú, simplemente que tengas algo de experiencia. También debes llevar tu propio equipo.

Qué aprenderás

Aprenderás a crear aplicaciones desde cero con Angular 4 y TypeScript. Verás un montón de buenas prácticas y patrones muy valorados profesionalmente. También aprenderás conceptos que te servirán en tecnologías similares, como inyección de dependencias, testing unitario y end to end, TypeScript y trabajar con interfaces basadas en componentes.

Cómo se estructura

El curso es totalmente práctico. Durante las 8 sesiones, programarás una aplicación de reproducción de música. Te conectarás a la API de Spotify para obtener artistas, canciones y realizar una búsqueda instantánea al estilo Google. Durante las primeras 6 sesiones desarrollaremos juntos la aplicación y veremos todo lo que nos ofrece Angular 4. Durante las dos sesiones restantes desarrollaréis funcionalidad extra por equipos y tendréis la oportunidad de pedir profundizar sobre los temas qué más os interesan a la vez que resolveremos dudas.

Programa

Sesión Contenido
TypeScript La primera sesión estará enfocada a aprender TypeScript. Pero también introduciremos Angular, conoceremos sus puntos clave y veremos qué podemos hacer con él.
Angular Cli y estructura Montaremos la aplicación usando Angular Cli, haremos un diseño técnico de la aplicación y montaremos la pantalla de “Login”. Veremos cómo montar un sistema de autenticación oAuth completo con Angular 4
Homescreen Durante esta sesión crearemos la home screen. Para ello crearemos varios componentes reusables y los servicios de API. Veremos cómo extender el servicio Http de Angular y usar la configuración de entorno correctamente para crear las peticiones a la API de spotify. También trabajaremos con el router y rutas anidadas.
Player y estado Esta sesión estará centrada en la gestión del estado con ngrx (similar a Redux). Veremos qué es el estado de la aplicación y qué ventajas tiene usar una librería para su gestión, así como todas las buenas prácticas. Pondremos todo en práctica montando el reproductor de música.
Artista y búsqueda Crearemos la vista de artista. Reaprovecharemos componentes y trabajaremos de nuevo con las rutas y la API.
También desarrollaremos una búsqueda instantánea (estilo Google) en la que trabajaremos activamente con el estado, las rutas dinámicas y una característica clave de RxJS: Cancelar peticiones xhr.
Testing unitario y e2e Durante esta sesión veremos cómo testear nuestra aplicación Angular, qué es el TDD, los test E2E y por qué es importante.
Al hacer los test entenderemos el porqué de muchos conceptos que hemos visto, como la inyección de dependencias.
Funcionalidad extra por equipos (2 sesiones) Durante estas dos sesiones el grupo desarrollará funcionalidad nueva por equipos sobre la aplicación. En estas dos sesiones trabajaremos de forma colaborativa en el proyecto mediante GIT para poner en práctica lo aprendido y que surjan dudas y problemas nuevos que resolveremos en grupo.
También se pueden organizar charlas de temas de interés para el grupo. Incluyendo otras tecnologías como React o Vue, que tienen cierta similitud a Angular como preguntas frecuentes en entrevistas técnicas.
